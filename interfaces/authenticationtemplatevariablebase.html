Optional mock

mock: undefined | string
  • Defined in lib/auth/templates/types.ts:168

The BTL-encoded test value for this variable. This test value is used during development and can provide validation when importing this template into a new system.

When testing, all variables for all entities are initialized to their mock and each unlocking script is tested against the locking script, ensuring it is able to unlock it. For inline scripts, variables are also initialized to their mocks when evaluating inline script tests.

Note, mock is itself defined in BTL syntax, but mock scripts do not have access to evaluations, other variables, or scripts. (Hex, BigInt, and UTF8 literals are permissible, as well as push notation and comments.)
